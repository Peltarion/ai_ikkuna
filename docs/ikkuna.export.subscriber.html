ikkuna.export.subscriber.ratio module¶

class ikkuna.export.subscriber.ratio.RatioSubscriber(kinds, tag=None, subsample=1, ylims=None, backend='tb', absolute=True)¶

Bases: ikkuna.export.subscriber.subscriber.PlotSubscriber

A ikkuna.export.subscriber.Subscriber which computes the average ratio between two quantities. The dividend will be the first element of the ikkuna.export.subscriber.Subscription.kinds property, the divisor the second. Therefore it is vital to pass the message kinds to the ikkuna.export.subscriber.Subscription object in the correct order.

_abc_negative_cache = <_weakrefset.WeakSet object>¶
_abc_negative_cache_version = 197¶
_abc_registry = <_weakrefset.WeakSet object>¶
_metric(module_data)¶

The ratio between the two kinds is computed over the subset of not-NaN values and added to the record.

ikkuna.export.subscriber.subscriber module¶

This module contains the base definition for subscriber functionality. The ikkuna.export.subscriber.Subscriber class should be subclassed for adding new metrics.

class ikkuna.export.subscriber.subscriber.Subscription(subscriber, kinds, tag=None, subsample=1)¶

Specification for a subscription that can span multiple kinds and a tag.

_tag¶

str – Tag for filtering the processed messages

_subscriber¶

ikkuna.export.subscriber.Subscriber – The subscriber associated with the subscription

counter¶

dict(ikkuna.utils.NamedModule or str, int) – Number of times the subscriber was called for each module label or meta data identifier. Since one ikkuna.export.subscriber.Subscriber is associated with only one configuration of ikkuna.export.messages.ModuleData, this will enable proper subsampling of message streams.

kinds¶

list(str) – List of string identifiers for different message kinds. These are all the message kinds the subscriber wishes to receive

_subsample¶

int – Factor for subsampling incoming messages. Only every subsample-th message will be processed.

_handle_message(message)¶

Process a newly arrived message. Subclasses should override this method for any special treatment.

class ikkuna.export.subscriber.subscriber.SynchronizedSubscription(subscriber, kinds, tag=None, subsample=1)¶

Bases: ikkuna.export.subscriber.subscriber.Subscription

A subscription which buffers messages and publishes a set of messages, each of a different kind, when one round (a train step) is over. This is useful for receiving several kinds of messages in each train step and always have them be processed together.

_handle_message(message)¶

Start a new round if a new sequence number is seen.

_new_round(seq)¶

Start a new round of buffering, clearing the previous cache and resetting the record for which kinds were received in this round.

Parameters:seq (int) – Sequence number for the new round
Raises:RuntimeError – If not all desired kinds have been received yet in the current round
