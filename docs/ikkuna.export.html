class ikkuna.export.export.Exporter¶

Bases: object

Class for managing publishing of data from model code.

An Exporter is used in the model code by either explicitly registering modules for tracking with Exporter.add_modules() or by calling it with newly constructed modules which will then be returned as-is, but be registered in the process.

e = Exporter(...)
features = nn.Sequential([
    nn.Linear(...),
    e(nn.Conv2d(...)),
    nn.ReLU()
])

Modules will be tracked recursive unless specified otherwise, meaning the following is possible:

e = Exporter(...)
e.add_modules(extremely_complex_model)
# e will now track all layers of extremely_complex_model

Exporter.epoch_finished() should be called upon finishing an epoch.

No further changes to the model code are necessary, but for a call to Exporter.set_model() to have the Exporter wire up the appropriate callbacks.

class ikkuna.export.messages.ModuleData(module, kinds)¶

Bases: object

Data object for holding a set of artifacts for a module at one point during training. This data type can be used to buffer different kinds and check whether all expected kinds have been received for a module.

_module¶

str – Name of the layer

_expected_kinds¶

list(str) – The expected kinds of messages per iteration

_data¶

dict(str, torch.Tensor) – The tensors received for each kind

_seq¶

int – Global sequence number of this class (incremented whenever a ModuleData is created)

_step¶

int – Sequence number (training step) of the received messages (should match across all msgs in one iteration)

_epoch¶

int – Epoch of the received messages (should match across all msgs in one iteration)
